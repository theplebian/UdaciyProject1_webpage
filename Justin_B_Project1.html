What is HTML?

HTMLis an acronym for Hyper T ext Markup Language. HTML provides the basic underlying structure and static content of web pages. To use the internet a user makes a request, which is interpreted using http(hyper text transfer protocol) which is sent and interpreted by a special computer called a server. A server responds to the request and replies via the internet. The results are rendered by the user's internet browser such as Microsoft Explorer, Safari, or Google Chrome.

The internet content that you see in your browser is divided into a number of html elements by the webpage designer. Elements are generally delineated by an opening and a closing html tag. The tags are important because html code is just some typed in plain text that is read by the browser. Plain text does not have the formatting or organizational options that you find in your word processor. In plain text their are no tables, boxes, different fonts, or the ability to make text bold or italic. Tags in html tell the browser how to interpret content.

There are a few tags that stand alone without a closing tag. These tags are called void tag . A good example of a void tag is a tag that directs the browser to an image. Since images are not plain text, the image tag in html tells the browser where to get the image to display in the given area of the page.

Web Page Organization

HTML automatically divides content into blocks. Each block is automatically given a certain amount of space around it. Some elements, such as paragraphs exist in their own block of space. Others exist within that block. For example a block of paragraph text has space above and below it to separate it from other elements, while bold or italic text within the paragraph do not break from other text. Elements that are automatically allotted their own block of space are called block elements, while those that are not are called inline elements.Some examples of inline tags are the anchor tag <a>, the italics tag <em>, and the paragraph tag <p>. Some examples of block tags are the division tag <div>, the paragraph tag <p>, and the form tag <form>.

The html programmer can use division tags to group content, including multiple block elements. For example, the navigation menu to the right was created as a separate block of content using division tags. A division or block of elements can be referred to as a parent element, and a sub-division or element within a division is called the child element of that division or element. A tree diagram or flow chart is sometimes made to illustrate the dependence of various organizational elements, known as a DOM (Document Object Model) for the webpage. The styling and positioning of elements on a web page can be refined by using CSS.

What is CSS?

CSS stands for Cascading Style Sheets. CSS is used to refine the position and styling of elements on a page. The divisions or classes defined in html can be used by CSS to apply styling. Styling defined for parent elements is automatically inherited by child elements. Thus styling instructions are said to cascade like a waterfall from parent elements to their child elements.

CSS instructions can be embedded within the header of the HTML document, inline with the elements themselves, or written in a separate document. Normally, styling instructions are written in a separate document that is loaded with the HTML code by the browser by a link provided in the header of an HTML document. This allows for the most efficient use of CSS. By writing the styling separately using content divisions, html code is less cluttered and the necessity of styling each individual element is avoided, embedded styling features must be repeated for every element needing that style. In contrast, when a separate CSS file is created styling can easily be applied to classes or divisions of elements . Styling features of parent elements are automatically inherited by child elements. Thus, repetition is avoided.

The header of an HTML document contains instructions that are not displayed on the page, but tell the browser the title to display in the browser tab and other instructions that must be loaded in order to render the page correctly.

What are Programming Languages?

A programming language is like an interpreter that communicates between you and computer. Computer languages contain English commands that you can understand coupled with a grammar that facilitates communication with the computer. The written commands are compiled or in other words translated into computer language by a compiler.

Computers work best with a sort of arithmetic with some subtle differences from what you may have learned in algebra class. Consider in Python the command a = 5. In Python this is not a declarative statement, it is imperative. That is to say, in algebra class the statement declares that the value of "a" and 5 are identical, i.e. a is 5. Such a declaration is either true or false. However, in Python a = 5 means no matter what the value of "a" might have been previously, from now on when I say "a" I mean five. It is a command. In older computer languages, this was made explicit by the preceeding the equality with the word "let". So, to assign the value of 5 to the letter a, one would write, "let a = 5." In Python, if one wanted to compare two numbers insteading of assigning a value, then a double equal sign makes the declarative comparison, e.g. a==5. Writing a==5 in Python yields either a value of true or false.

Python and other languages can also work with strings of characters withot interpreting them as a mathematical expression. Such strings of characters, intended to be interpreted simply as text, are simply reffered to as strings However, some of the same grammar is applied to these as well. For e.g. arithmetically I can assign a = 5 and b = 2 so that the computer outputs 7 for the expression a+b. However, I can also write a = "Hello, " and b = "Tom.", so that the expression a+b would be interpreted as, "Hello, Tom." Adding strings together as in the previous example is called contenation.

Since, computers cannot text written by humans in their own language, programmers must help bridge the gap for users. To do this the programmers have to be able to refer to particular parts of a string. In the early days of personal computers interactive text games were common. The user would read the description of an environment just as in the typical novel, but then have the freedom to type a command for the main character to perform. The programmer then would tell the computer to interpret the sentence the user wrote by looking for verb and object in the sentence. Depending on the verb and object combination would then result in the program outputting text that continues the story. In Python one could find the location of particular bits of string with the command find. For example, if the user_input ="attack the ogre", the position of the verb "attack" in a string could be found in Python with the command, user_input.find("attack"). The computer then begins at the beginning of the string counting the first character as zero and then moving on until the word, "attack" is found. In this case, the expression would be interpreted as the number zero since it is found at the very beginning of a string. The computer can also be told to start looking at some other place than the beginning and select specific parts of what the user inputted.

To avoid repetition a programmer might want to define a procedure that can be repeated. When a programmer does this in Python it looks very much like the function notation that you used in your algebra class. For instance, to convert degrees celsius to fahrenheit, in algebra class we could write the formula f(c)= 1.8*c+32. In the algebraic expression the function is named f for degrees fahrenheit the letter c on the left of the equation in parenthesis indentifies the input of the function. The letter c was chosen for degrees celsius. In Python, we could accomplish the same thing by writing:
     def f(c):
          return 1.8*c+32
Now, if one wanted the computer to convert degrees 40 degrees celsius to degrees fahrenheit, one would write: print f(40). Programmers my refer to such a function as a method

Every programming language has a few basic elements that make it function. In fact there are suprisingle few structures needed to make any program. The logical structures required are sequence, grammar, conditions, and loops. Sequence means that the computer reads the lines of code in order. Grammar is the way code is written so that it is interpreted by the computer correctly. Grammar includes things like when to capitalize, insert commas, what type of parenthesis to use, etc. Grammar varies between programming languages. Conditional statements contain a condition that triggers the execution of a block of code when ever the condition is true. In Python all conditional statments begin with the word if. A loop is a block of code that is repeated. In Python there are while loops that repeat a block of code as long as (while) a certain condition is met, and for loops that perform a block of code for every element in a given list.
